# Regulations Management System Setup

## Backend Setup (Go)

### Prerequisites
- Go 1.24.5+
- MySQL running locally

### Database Setup

1. Create a database named `Management_Portal`:
```sql
CREATE DATABASE Management_Portal;
```

2. The `regulations` table will be created automatically when you run the server.

### Run Backend

1. Navigate to the server directory:
```bash
cd server
```

2. Download dependencies:
```bash
go mod download
```

3. Run the server:
```bash
go run main.go
```

The backend will start at `http://localhost:8080`

### API Endpoints

- **GET** `/api/regulations` - Fetch all regulations
- **POST** `/api/regulations/create` - Create a new regulation
- **DELETE** `/api/regulations/delete?id={id}` - Delete a regulation

## Frontend Setup (React)

### Prerequisites
- Node.js 14+
- npm or yarn

### Run Frontend

1. Navigate to the client directory:
```bash
cd client
```

2. Install dependencies:
```bash
npm install
```

3. Start the development server:
```bash
npm start
```

The frontend will open at `http://localhost:3000`

## Features

### Login Page
- Username: `admin`
- Password: `987654321`
- Navigates to `/dashboard` on successful login

### Regulations Page
- View all regulations in card format
- Add new regulations with name and academic year
- Delete regulations
- Real-time updates from backend

## Database Table Schema

The `regulations` table has the following structure:
```sql
CREATE TABLE regulations (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  academic_year VARCHAR(50) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

## Notes

- Make sure MySQL is running before starting the backend
- The backend runs on port 8080
- The frontend runs on port 3000
- CORS is enabled for cross-origin requests
